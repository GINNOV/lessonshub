# Amministrazione e Operazioni

Gli amministratori hanno una visione d'insieme della piattaforma tramite la [dashboard admin](/dashboard).

## Aree di Gestione Principali

- **Utenti:** Invita nuovi utenti, sospendi account o modifica profili. Puoi anche "Mettere in pausa" un utente (uguale alla funzione "Prenditi una pausa").
- **Lezioni:** Supervisiona l'intero catalogo delle lezioni, esegui migrazioni del database o inizializza i template predefiniti.
- **Email:** Monitora i log di tutte le email inviate (transazionali e di marketing).
- **Impostazioni:** Configura impostazioni globali, valori dei punti e flag delle funzionalità.
- **Automation:** Gestisci job AI ricorrenti e accessi di automazione basati su token.
- **Cron:** Attiva manualmente i job di sistema (come le notifiche di "Data di Inizio") o simula orari per i test.
- **Profilo:** Visualizza il tuo profilo con controlli amministrativi extra.

## Job di Sistema (Cron)

LessonHub esegue job automatici per mantenere tutto in movimento:
- **Ogni ora:** Invia notifiche di "Data di Inizio" (eseguito giornalmente alle 10:00 UTC su alcuni piani).
- **Giornaliero (09:00 UTC):** Invia promemoria dei compiti e riepiloghi settimanali.

**Test:** Puoi usare la pagina **Cron Test** per simulare orari specifici e assicurarti che le email partano correttamente.

## Database e Manutenzione

- **Migrazioni:** Esegui sempre tramite gli script `npm` forniti per garantire la sicurezza.
- **Seeding:** Se mancano template predefiniti o badge, lo script di seed li ripristina senza cancellare i dati esistenti.
- **Token di automazione:** Dopo aver aggiornato il codice relativo all'automazione, esegui l'ultima migrazione Prisma prima di emettere token o chiamare le nuove API.

## Automazioni Codex Per La Creazione Delle Lezioni

LessonHub supporta ora token di automazione concessi dall'amministratore, così le automazioni Codex possono creare lezioni tramite chiamate API server-to-server senza usare una sessione browser.

### Configurazione Admin

1. Apri **Admin > Automation Tokens**.
2. Scegli l'account insegnante che dovrà risultare proprietario delle lezioni create dall'automazione.
3. Crea un token e copialo subito. Il valore completo viene mostrato una sola volta.
4. Salva il token in `/Users/mario/code/LessonHub/lessonhub-app/.env.automation` come `AUTOMATION_TOKEN`. Non inserirlo in documenti, chat o screenshot condivisi.

### File Locale Dei Segreti

Usa `.env.automation` nella root del progetto per i segreti e i default dell'automazione Codex:

```env
AUTOMATION_TOKEN=your_real_token_here
DEFAULT_CLASS_ID=your_class_cuid_here
DEFAULT_CLASS_NAME=Fall 2025
```

- `AUTOMATION_TOKEN` è obbligatorio.
- `DEFAULT_CLASS_ID` è consigliato se l'automazione deve assegnare sempre alla stessa classe.
- Se `DEFAULT_CLASS_ID` non è presente, l'attuale flusso di test da terminale chiama la API remota di contesto dell'automazione e usa come fallback la classe attiva dell'insegnante chiamata `Fall 2025`.
- `DEFAULT_CLASS_NAME` può essere cambiato se vuoi un fallback basato sul nome diverso.

### Endpoint Supportati

- **Lezioni standard:** `POST /api/automation/lessons`
- **Lezioni multi-choice:** `POST /api/automation/lessons/multi-choice`

Entrambi gli endpoint richiedono:

- `Authorization: Bearer <token>`
- `Content-Type: application/json`
- un campo `topic`, così l'automazione ha un soggetto stabile per la lezione generata

### Esempio Lezione Standard

```bash
curl -X POST https://your-domain.com/api/automation/lessons \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "topic": "Restaurant English",
    "assignmentText": "Answer in complete sentences.",
    "questions": [
      {
        "question": "How would you ask for the bill?",
        "expectedAnswer": "Could I have the bill, please?"
      }
    ],
    "difficulty": 2,
    "assignment": {
      "studentIds": ["student_cuid_here"],
      "classIds": ["class_cuid_here"],
      "startDate": "2026-03-27T13:00:00.000Z",
      "deadline": "2026-03-29T21:00:00.000Z",
      "notificationOption": "immediate",
      "reassignExisting": false
    }
  }'
```

### Esempio Lezione Multi-Choice

```bash
curl -X POST https://your-domain.com/api/automation/lessons/multi-choice \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "topic": "Travel vocabulary",
    "questions": [
      {
        "question": "Which phrase asks for directions?",
        "options": [
          { "text": "Where is the station?", "isCorrect": true },
          { "text": "I need a taxi.", "isCorrect": false }
        ]
      }
    ],
    "difficulty": 3,
    "assignment": {
      "classIds": ["class_cuid_here"],
      "notificationOption": "on_start_date",
      "reassignExisting": true
    }
  }'
```

### Note Sul Payload

- `title` è opzionale. Se omesso, LessonHub usa `topic` come titolo.
- `difficulty` deve essere un intero da `1` a `5`.
- `price` è opzionale e per default vale `0`.
- `assignment_notification` e `scheduled_assignment_date` funzionano come nelle API delle lezioni usate dagli insegnanti.
- Le lezioni standard richiedono `assignmentText` e almeno una domanda.
- Le lezioni multi-choice richiedono almeno una domanda, almeno due opzioni per domanda ed esattamente una risposta corretta.
- Il blocco opzionale `assignment` permette all'automazione di colpire specifici `studentIds`, `classIds`, oppure entrambi.
- Se `assignment` viene omesso, LessonHub usa il comportamento di auto-assegnazione definito a livello di lezione.
- `assignment.notificationOption` supporta `immediate`, `on_start_date` oppure `none`.
- `assignment.reassignExisting` resetta i compiti esistenti per gli studenti selezionati invece di saltarli in silenzio.
- `assignment.startDate` e `assignment.deadline` vengono applicati a ogni studente risolto in quella chiamata.
- Un default pratico è inviare `assignment.classIds` usando `DEFAULT_CLASS_ID` preso da `.env.automation`.
- Nell'attuale script di test, le lezioni vengono assegnate automaticamente alla classe configurata. Se non è configurato un class id, lo script risolve la classe tramite `GET /api/automation/context` e cerca la classe attiva dell'insegnante chiamata `Fall 2025`.

### Regole Di Sicurezza

- I token sono associati a uno specifico account insegnante.
- I token vengono salvati in forma hash, possono essere revocati dal pannello admin e registrano l'ultimo utilizzo.
- Se un token viene esposto, revocalo subito e genera un nuovo token.
- Mantieni i token di automazione separati dalle chiavi AI personali o dai segreti dell'app in `.env.local`.

## Automazioni Giornaliere Integrate

LessonHub può ora generare lezioni standard giornaliere direttamente dentro l'app usando la chiave Gemini configurata e il flusso cron esistente su Vercel.

### Come Funziona

- Crea uno o più job da **Admin > Automation**.
- Ogni job punta a un insegnante e a una classe attiva.
- La route cron giornaliera esegue i job abilitati una volta al giorno.
- Ogni job salva la propria cronologia di esecuzione e non crea una seconda lezione per lo stesso giorno.

### Cosa Puoi Controllare

- Nome del job
- Insegnante
- Classe
- Stato abilitato o disabilitato
- Difficoltà
- Prezzo
- Prompt opzionale per guidare Gemini
- Pool opzionale di temi per controllare la rotazione degli argomenti

### Operatività

- Usa **Run now** nella pagina Automation per testare i job immediatamente.
- Usa **Admin > Cron** per attivare manualmente l'automazione giornaliera delle lezioni con una data di riferimento opzionale.
- I job giornalieri dipendono da una chiave Gemini configurata nelle impostazioni AI admin.

## Sicurezza e Best Practices

- **Segreti:** Tutte le chiavi sensibili (password database, chiavi API) sono conservate in `.env.local` e mai condivise.
- **Dati:** I valori finanziari (come i prezzi) sono gestiti con alta precisione.
- **Classifiche:** I risparmi sono visualizzati chiaramente con codici colore (Verde per positivo, Rosso per negativo) per evitare confusione.

## Playbook di Supporto

Compiti amministrativi comuni:
- **Accesso Hub Guide:** Se uno studente sostiene di aver pagato ma non vede le guide, controlla lo stato del pagamento nel suo Profilo.
- **Errori Testi:** Se gli insegnanti segnalano problemi con le lezioni con testo, spesso significa che è necessaria una migrazione del database.
- **Fusi Orari:** Incoraggia tutti gli utenti a impostare il proprio fuso orario. Fusi orari errati sono la causa numero 1 di scadenze "mancate".
